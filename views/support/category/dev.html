Modes de récupération

Admin Services dispose de deux systèmes de récupération en cas de défaillance.

  • Mode de récupération : Il s'agit d'un mode de récupération automatique qui se déclenche en cas d'erreur critique empêchant le bon fonctionnement d'Admin Services. Il peut être utilisé manuellement et contient diverses options de récupération.
  • Mode sans échec : Le mode sans échec est accessible uniquement au démarrage d'Admin Services et désactive tout ce qui pourrait ralentir ou bloquer le navigateur, poser des problèmes, etc.

Les différents modes proposés sont accessibles à tous, mais sont surtout utilisés par les développeurs d'applications.

Mode de récupération

Il existe deux méthodes d'accès :

  • Automatiquement, lors d'un incident au démarrage : sauvegardes de session corrompues, profil utilisateur corrompu, etc.
  • Manuellement, dans les paramètres.

Une fois le mode de récupération démarré, vous disposez de plusieurs options :

  • Réinitialiser le profil (efface toutes les données du profil)
  • Mettre à jour le profil (met à jour les données du profil, permet de réparer le compte en cas de corruption)
  • Réinitialiser Admin Services (efface toutes les données d'Admin Services)
  • Réinitialiser le système de démarrage (réinitialise les paramètres de démarrage)
  • Démarrer Admin Utility (mode sans échec)
  • Télécharger la dernière mise à jour stable via GitHub (télécharge la dernière version d'Admin Services)
  • Quitter et passer sur Admin Services (quitte le mode de récupération)

Mode sans échec

Le mode sans échec est un mode de récupération qui permet de continuer à accéder à Admin Services.

Pour y accéder, appuyez sur la touche S au démarrage d'Admin Services.

Note : Si un message ne s'affiche pas après avoir appuyé sur la touche, cela signifie que le mode sans échec n'a pas été chargé.

Le mode sans échec :

  • Désactive les effets de transparence
  • Désactive les animations
  • Désactive les effets de flou
  • Désactive les effets de transition
  • Désactive les effets de défilement
  • Les applications tierces ne sont pas chargées

Note : les applications tierces sont désactivées, sauf pour les comptes développeur enregistrés. Cependant, le compte développeur ne déverrouille que les applications publiées sur le compte concerné.
